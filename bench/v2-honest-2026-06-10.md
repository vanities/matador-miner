# Honest per-patch A/B — v2 nonce-seed mode (2026-06-10)

**This is the bench that represents live mining.** The earlier numbers in
`patches-ab-2026-06-09.md` were taken in the bench's *default v1 mode*, which
reuses one seed and caches matrix generation — so it measured mostly the matrix
product and overstated the live gain (~3.7×). Post-125,000 mining is v2
nonce-seed: every nonce derives a fresh seed and regenerates both 512×512
matrices, so matrix-gen (SHA-256) can't be cached and dominates.

Reproduce: `make bench` (or `bash bench/ab.sh`). Flags:
`--block-height <tip> --nonce-seed-height 0 --product-digest-height 0`.
RTX 5090, miner paused, `n=512 b=16 r=8`, 100 iters, median nonces/s.

## Reading this

- **Real live gain from all five patches: 2.79×** (not the 4.48× the v1 bench
  showed). Still a large, real win — and every patch is byte-exact-validated
  and consensus-clean live (zero CPU-confirm mismatches). **Nothing needs
  redoing.**
- **The two big movers are matrix-gen** (windowed SHA + single-reduction: +69%)
  **and factored compression** (+55%) — both on the per-nonce digest path that
  live mining actually exercises.
- **Scanner-only patches are small live** (windowed SHA +2.6%, midstates folded
  into the +3.6% round-3 step): the scanner already has throughput headroom over
  the digest, so making it faster barely moves the bottleneck. They were worth
  more in the v1 bench (and are already upstreamed in PR #58), but live, the
  digest is the limiter.

## Live ground truth (not the bench)

Bench numbers are uncontended single-process. The live miner runs alongside the
full node (p2p, validation, CPU-confirm, template refresh), so its sustained
rate is lower than the bench. Measure it with `make stats` (counter-delta meter)
or the profile counter
`getmatmulchallengeprofile .service_profile.runtime_observability.solve_pipeline.batched_nonce_attempts`.
Real network share = **blocks won ÷ ~960/day** (90 s target spacing), which is
**~0.3–0.4%** — *not* bench-nonces/s ÷ networkhashps (mismatched units).

## Next lever

Live is matrix-gen-SHA-bound. The seed is constant across all 262,144 elements
of a matrix, so the first 8 SHA rounds are recomputed redundantly — a
seed-midstate hoist (~12.5% of matrix-gen) is the next byte-exact target. Bench
it with `make bench` in this same v2 mode.

## Round 4 (later 2026-06-10): matrix-gen seed-midstate

The matrix-gen SHA hashes `seed||index` — the 32-byte seed fills w[0..7], so
SHA rounds 0-7 are identical across all 262,144 elements of a matrix. Hoisting
them is the obvious win, but it has a trap: a shared-memory hoist (thread 0
computes the midstate, `__syncthreads`, all threads read it) was **3× SLOWER** —
the per-block barrier + passing the midstate as array params (which CUDA spills
to local memory) cost far more than the 8 rounds saved on this barrier-free,
register-tight kernel. The **2-kernel form wins**: a tiny precompute kernel
writes each seed's midstate (16 words) to global; the gen kernel loads them as
scalars (registers) and resumes at round 8. No barrier, no local-memory arrays.

| `btx-miner:prev4` | 5 | 155,047 | 399 | 389 |
| `btx-miner:local` | 6 — + seed-midstate | **163,699** | 385 | **425** |

+5.6% end-to-end (and +9% efficiency). Isolated matrix-gen kernel +12.4%,
byte-exact over 2.1M elements. From clean 0.32.3: **54,964 → 163,699 = 2.98×**.

## Things tried and rejected (negative results)

- **Shared-memory midstate hoist** — 3× slower (barrier + local-mem array params).
  Superseded by the 2-kernel form above.
- **Pipeline config knobs** (`--async 1`, `--batch-size 4096/8192`,
  `--prefetch-depth 2/3`, `--prepare-workers 8`, `--pool-slots 8`): all within
  noise (≤+1.2%) in v2 mode. The live pipeline is already GPU-bound and
  well-saturated; defaults kept. See `bench/config-sweep.sh`.

## Noted but not taken

- **Extend the midstate to w[16..22]** (the 7 seed-constant schedule words, also
  retry==0-only): ~+2% more, but doubles the consensus-path surface for a small
  gain — left for a reviewed change, not an overnight one.
- **Tensor-core M31 GEMM**: ~2× theoretical on the product, but the product is
  already factored to ~12.6M MACs and matrix-gen (SHA) now dominates, so the
  ceiling is low for high risk. Parked.
